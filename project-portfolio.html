05 — Design & Development

Personal UX
Portfolio Website

Designed and built this portfolio site from scratch — two distinct portfolios for two distinct audiences, each with its own visual language, inner project pages, and a shared contact page. A full design + development process in one project.

Role Designer & Developer
Stack HTML, CSS, JavaScript, Formspree
Hosting GitHub Pages (Free)
Deliverables 2 portfolios, 7 inner pages, 1 contact page
01 — The Problem

Two audiences. Two portfolios. One designer.

The challenge was unusual: I needed to present myself credibly to two completely different hiring audiences — UX and product design teams who care about research rigor and design thinking, and data and analytics teams who care about analytical skills and business insight.

A single generic portfolio would undersell both. The solution: two separate portfolio sites with distinct visual identities, each tailored to its audience — while still feeling like they come from the same person.

"Your portfolio is itself a UX project. Who are the users? What are their goals? What do they need to see in the first 10 seconds to keep reading?"

02 — Design Strategy

Distinct identities, shared values

Before writing a line of code, I spent time defining what each portfolio needed to communicate at a gut level — not just in content, but in aesthetic feel.

  • UX
    Dark, refined, editorial — signals design sophistication, attention to craft, and contemporary visual sensibility. Dark backgrounds read as intentional and confident in design circles. Serif + mono type pairing communicates a mix of humanity and precision.
  • DATA
    Light, structured, authoritative — signals clarity of thought, professional maturity, and clean communication. An off-white editorial palette with sharp red accents communicates analytical rigor without feeling corporate or cold.

Both sites share the same underlying structural logic — hero, projects, about, contact — but the visual expression, typography, and copy voice are tuned for each audience independently.

03 — Technical Build

No frameworks. Just HTML, CSS, and JS.

The decision to build without a framework was intentional. Employers reviewing a UX candidate's portfolio code expect to see clean, semantic HTML and well-organized CSS — not a React app that obscures whether you understand the fundamentals.

  • 01
    Shared CSS architecture A single style.css file powers all inner pages for the UX portfolio — CSS custom properties (variables) ensure visual consistency without repetition across 7+ pages.
  • 02
    Mobile-first responsive design All layouts built mobile-first with CSS Grid and Flexbox. No media query hacks — clean breakpoints at 768px handle the desktop expansion.
  • 03
    Formspree contact integration The contact form posts to Formspree — no backend required, no server costs. Submissions arrive via email and are logged in a Formspree dashboard.
  • 04
    GitHub Pages hosting Both portfolios hosted free on GitHub Pages — giving employers a GitHub profile to review alongside the work, adding credibility in technical hiring contexts.
