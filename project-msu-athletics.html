01 — Usability Research & Redesign

MSU Athletics
Website Redesign Study

A full usability evaluation of MSU's Athletics site — from heuristic analysis and user task testing through to redesigned navigation flows that improve accessibility and clarity for every visitor.

Role UX Researcher & Designer
Methods Heuristic Evaluation, Usability Testing, SUS
Tools Figma, Google Forms, QDA Miner
Context MSU — Evaluating Human-Centered Technology
01 — The Problem

A high-traffic site with hidden friction

MSU Athletics serves thousands of fans, students, staff, and prospective student-athletes — each with distinct goals. The existing site had grown organically over years, creating a navigation structure that worked well for insiders but left new users stranded.

The core question: where exactly does the site break down, and for whom? The goal was to move beyond subjective opinion and ground redesign decisions in real user behavior and measurable data.

"Users consistently knew what they were looking for — tickets, schedules, rosters — but couldn't find them. The information architecture was the culprit, not the content."

02 — Research Approach

Two-phase evaluation: experts first, users second

The study began with a heuristic evaluation using Nielsen's 10 usability heuristics. This let me identify structural issues quickly and cheaply before involving participants, ensuring usability tests would focus on the most impactful areas.

  • 01
    Heuristic Evaluation Systematically assessed the site against 10 Nielsen heuristics, rating severity on a 0–4 scale. Flagged 14 issues, with navigation consistency and error prevention rated most critical.
  • 02
    Task-Based Usability Testing Recruited 5 participants representing different user archetypes — casual fan, prospective athlete, student, and staff. Each completed 6 structured tasks while thinking aloud.
  • 03
    SUS Scoring & Survey Collected System Usability Scale scores post-test and supplemented with open-ended survey questions to capture emotional responses alongside task metrics.
  • 04
    Affinity Mapping Synthesized all qualitative notes into themes, grouping pain points into navigation, search, mobile, and content hierarchy buckets.
03 — Key Findings

Where users got stuck

The SUS average came in at 58/100 — below the industry benchmark of 68, placing it in the "poor" range. Qualitative data told the same story in vivid detail.

58

Average SUS Score

Below the 68-point "okay" threshold — users consistently rated the experience as difficult.

4/6

Tasks with Navigation Issues

Most failed or delayed task completions traced back to unclear menu labels and inconsistent nav placement.

14

Heuristic Violations Found

Spread across visibility of system status, consistency, and error prevention — all core to daily use.

3

Critical Severity Issues

Three violations rated severity 4 (catastrophic): global search failure, mobile nav collapse, and roster lookup.

A recurring theme in think-aloud sessions: users expected the homepage to surface current game schedules and ticket links prominently. Instead, these were buried 2–3 levels deep — a mismatch between user mental models and site architecture.

04 — Design Recommendations

Evidence-backed redesign decisions

Each recommendation maps directly to a specific finding — no guesswork, no aesthetic opinions without data to back them up.

  • ↳
    Restructure the primary navigation Flatten the hierarchy from 4 levels to 2 for the most-accessed sections. Rename ambiguous labels ("Spartan Experience" → "Fan Guide") based on card sorting insights.
  • ↳
    Surface schedules & tickets on the homepage Add a persistent "Today / Upcoming" widget above the fold that updates automatically — eliminating the #1 pain point across all user types.
  • ↳
    Fix global search Implement a search bar in the fixed header with autocomplete and sport-specific filtering. The current search consistently returned irrelevant results.
  • ↳
    Redesign mobile navigation Replace the collapsed hamburger-within-hamburger pattern with a bottom nav bar for the 5 most-visited sections on mobile.
  • ↳
    Standardize roster page templates Create a consistent template across all sports — currently each sport page has a different layout, confusing users who navigate between them.
05 — Reflection

What I learned

This project reinforced that users rarely fail because the content is missing — it's almost always a discoverability problem. The information was there; the architecture hid it.

Running the heuristic evaluation before user testing also proved valuable. It sharpened the task design — I knew exactly which flows to probe — and made the user sessions far more efficient and revealing.

If I were to extend this project, I'd prototype the new navigation structure in Figma and run a second round of moderated testing to validate that the changes actually moved the SUS needle before any development work began.
